# 1. シナリオA：分断モデルにおけるホモ・エコノミカス的挙動
- 医療・介護連携が存在しないシナリオAは、連携欠如がもたらす負の連鎖を明確に示している。患者の転帰に関して、シナリオAは「Mean SDH Risk（社会的決定要因リスク）」および「Mean Isolation（孤立度）」において最も顕著な悪化を示し、結果として「Cumulative Acute Events（急性期イベント累積数）」も最大値を持続した。

-  一方で注目すべきは、提供者側の指標である「Mean Provider Fatigue（提供者の疲労）」および「Burnout Count（バーンアウト数）」の推移である。シナリオAではシミュレーション中盤までこれらの数値が他シナリオと比較して低く維持される。これは提供者の処理能力の高さを示すものではなく、管轄外の業務（退院後のフォローアップや生活支援等）に関与せず介入を打ち切ることで、個人のリソースが温存されている状態（FC-A1/A2の発現）を示唆している。すなわち、限定合理性に基づくホモ・エコノミカス的な力学が働き、患者の潜在的リスクを見過ごすことと引き換えに見かけ上の負担減が生じていると言える。

# 2. シナリオB：利他性の過剰依存に伴うネットワークの崩壊
- 本研究の重要な知見の一つは、共同体メカニズムに依存したシナリオBの動態である。シナリオBは、患者の健康指標の改善や急性期依存の低下においてシナリオCと同等の良好な結果を達成した（comparison_extended）。しかし、その代償は少数の提供者への過度な負荷として現れる。
- シナリオBでは序盤から「Mean Provider Fatigue」が急増し、全シナリオ中で最も早期に最大値（0.9以上）に達している（comparison_main）。このネットワークの脆弱性は、「Milestone M3（連携密度）」の推移に象徴的に表れている。シナリオB（図中オレンジ線）はDay 30付近で連携密度が突如としてゼロに急落しており、これは利他的な特定のノード（提供者）に負荷が集中し、連鎖的な機能不全に陥る「FC-B3（共同体メカニズムの枯渇）」の発火を示している。結果として、個人の利他性に過度に依存したモデルは持続可能性に欠けることが実証された。

# 3. シナリオCおよびアブレーション解析：AI介入における負荷分散の不可欠性
- AI介入を伴うシナリオC、およびその構成要素を検証したアブレーション・スタディ（ablation_final, fc_heatmap）の比較から、システム介入における「負荷分散（N3）」の重要性が論理的に裏付けられた。
- まず、「Gini Fatigue（疲労のジニ係数）」の比較において、シナリオCは他条件に比べ提供者間の疲労の不平等を最も強力に抑制している（ablation_final）。さらに決定的な所見として、AI提案の受容性に関するヒートマップ解析（fc_heatmap）が挙げられる。完全なシナリオCにおいては「FC-C1（AI提案の拒絶）」の発生率が0.0に抑えられているのに対し、負荷再配分ナッジ（N3）を除外した「C-noN3」や、LLMのプロンプトのみに依存する「C-onlyL1」では、同指標が34.8〜35.0へと急激に上昇する。
- これは、システム側が「N5: 社会的処方」のような正当な支援策を提案したとしても、現場の負荷分散（N3）が並行して実行されなければ、疲弊した提供者によって「業務過多」として介入が拒絶されるという、医療現場における強力な動学（Dynamics）を示している。すなわち、AI介入の真価は単なる最適解の提示ではなく、リソースの動的な再配分による実行可能性の担保にある。

# 4. 限界（Limitation）：最適化の限界と構造的制約
- 本シミュレーションにおいては、AI介入を最適化したシナリオCであっても、「Milestone M4（提供者の回復）」および「M5（連携会議の定常化）」の達成率はほぼゼロにとどまった。
- この結果はモデルの欠陥を示すものではなく、本システムの限界を誠実に表している。すなわち、AIのナッジを通じたミクロ・メゾレベルの業務最適化や負荷分散だけでは、マクロレベルでの絶対的なマンパワー不足やリソースの枯渇という根本的な構造問題までは解決し得ないということである。本研究におけるAIの役割は、限られたリソース下におけるシステムの延命および破綻の遅延化として位置づけられるべきであり、抜本的な解決には人的資源の拡充や制度的アプローチの併用が不可欠である。
